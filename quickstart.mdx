<Note>
BBOT runs on **Linux only**. Python 3.9 or higher is required.
</Note>

## Install BBOT

<Steps>
  <Step title="Install via pipx (recommended)">
    `pipx` installs BBOT in an isolated environment and puts the `bbot` command on your PATH.

    ```bash
    pipx install bbot
    ```

    If you don't have `pipx`:
    ```bash
    pip install pipx
    pipx ensurepath
    ```
  </Step>
  <Step title="Verify the installation">
    ```bash
    bbot --version
    ```
  </Step>
  <Step title="Run your first scan">
    ```bash
    bbot -t evilcorp.com -p subdomain-enum
    ```
  </Step>
</Steps>

## Example scans

### Find subdomains

Passive API sources plus recursive DNS brute-force with NLP-powered subdomain mutations:

```bash
# Full subdomain enumeration
bbot -t evilcorp.com -p subdomain-enum

# Passive sources only (no active DNS)
bbot -t evilcorp.com -p subdomain-enum -rf passive
```

### Crawl a website

```bash
bbot -t evilcorp.com -p spider
```

### Harvest emails

```bash
bbot -t evilcorp.com -p email-enum

# Combine with subdomain enum and spider for maximum yield
bbot -t evilcorp.com -p email-enum subdomain-enum spider
```

### Web scanning

```bash
# Quick web scan
bbot -t www.evilcorp.com -p web-basic

# Thorough web scan
bbot -t www.evilcorp.com -p web-thorough
```

### Everything at once

```bash
bbot -t evilcorp.com -p kitchen-sink --allow-deadly
```

## Configure API keys

Many BBOT modules use third-party APIs. Add your keys to `~/.config/bbot/bbot.yml`:

```yaml ~/.config/bbot/bbot.yml
modules:
  shodan_dns:
    api_key: YOUR_KEY_HERE
  securitytrails:
    api_key: YOUR_KEY_HERE
  virustotal:
    api_key: YOUR_KEY_HERE
  github:
    api_key: YOUR_KEY_HERE
```

Or pass them directly on the command line:

```bash
bbot -t evilcorp.com -p subdomain-enum \
  -c modules.securitytrails.api_key=YOUR_KEY
```
